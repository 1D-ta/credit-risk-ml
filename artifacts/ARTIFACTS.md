# Artifacts Inventory

This directory stores runtime outputs generated by training, evaluation, inference, and monitoring steps.

## What is committed

To keep the repository lightweight and public-safe, only a minimal set of artifacts is committed:

- `artifacts/evidence/README.md`
- `artifacts/evidence/inference_success.json`
- `artifacts/evidence/inference_failure.json`
- `artifacts/evidence/temporal_training.log`
- `artifacts/evidence/drift_alert_rollback.log`
- `artifacts/models/.gitkeep`
- `artifacts/reports/.gitkeep`

## What is generated locally

The following are generated during local runs and are ignored by git:

- `artifacts/logs/`
- `artifacts/feature_stats/`
- `artifacts/monitoring/`
- model binaries under `artifacts/models/`
- JSON reports under `artifacts/reports/`

## Regeneration

Use Makefile targets to regenerate artifacts:

```bash
make train
make evaluate
make calibrate
make approve
```
